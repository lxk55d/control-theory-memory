# 差距分析：当前记忆系统 vs 控制论理想设计

## 一、现状快照

当前 Claude Code 记忆系统 = **一个扁平文件目录 + 一份索引文件**：

```
memory/
├── MEMORY.md          ← 索引（手动维护的一行一个链接）
├── environment-doc.md ← 记忆文件（带 frontmatter）
└── ...                ← 更多记忆文件
```

每条记忆的生命周期：**创建 → 永不改变 → 永不删除**（除非手动）。

## 二、逐项差距

### 🔴 缺失：反馈回路

| 理想设计 | 当前状态 | 影响 |
|---------|---------|------|
| 每条记忆记录访问次数、最后访问时间 | 没有任何跟踪 | 无法区分"活跃"和"僵尸"记忆 |
| 用户通过交互隐式提供反馈 | 无反馈采集 | 系统对用户需求变化完全盲目 |
| 检索结果根据历史表现调整排名 | 无排名机制 | 每次召回的排序都一样 |

### 🔴 缺失：遗忘调度

| 理想设计 | 当前状态 | 影响 |
|---------|---------|------|
| 连续衰减模型：d(retention)/dt = -k·retention | 无衰减 | 过时信息永远存在，消耗注意力 |
| 门控阈值：休眠区 → 摘要 → 清除 | 无层次化 | 只有"存在/不存在"的二元状态 |
| 重要记忆受保护（低衰减率） | 无区分 | 琐事和核心知识地位相同 |

### 🟡 弱：模型表达能力

| 理想设计 | 当前状态 |
|---------|---------|
| 向量嵌入（语义检索能力） | 无嵌入，纯文本匹配 |
| 关联网络（记忆之间的链接权重） | 仅 [[wikilink]] 占位，无边权重 |
| 聚类结构（知识簇） | 无显式聚类 |
| 动态重要性权重 | 重要性是静态的（hardcoded） |

### 🟢 已覆盖：基本结构

| 设计要素 | 状态 |
|---------|------|
| 文件作为记忆载体 | ✅ |
| frontmatter 元数据 | ✅ |
| 跨记忆链接（[[link]]） | ✅ |
| 索引文件 | ✅ |

## 三、差距等级

```
              当前系统                      理想设计
```

## 四、优先级排序

按"控制论价值 / 实现成本"排序，最大投入产出比在前：

| 优先级 | 改进项 | 控制论价值 | 实现成本 | 路径 |
|--------|-------|-----------|---------|------|
| **P0** | **遗忘控制器** | 核心——没有遗忘就没有调节 | 低——Python 脚本操作文件 | 连续衰减模型 + 门控阈值 |
| **P1** | **反馈观测层** | 核心——没有观测就没有控制 | 低——为 memory 文件增加元数据 | 访问计数 + 时间戳 |
| **P2** | **层级存储（活跃/休眠/存档）** | 高——分层是控制的基础 | 中——目录重组 + 摘要生成 | 三区架构 |
| **P3** | **自适应参数调优（PID）** | 高——系统自动收敛到最优 | 中——需积累反馈数据 | PID 控制器 |
| **P4** | **向量语义检索** | 中——提升召回质量 | 高——需嵌入模型 | slerp 或本地模型 |
| **P5** | **关联网络权重** | 中——结构涌现的基础 | 高——需图存储 | PageRank + Hebb学习 |
| **P6** | **漂移检测 + 结构涌现** | 最高——但依赖下层 | 高——需所有下层数据 | 聚类 + 贝叶斯断点 |
